# اینجا۳دی (INJA3D) — فروشگاه کامل، متصل به ووکامرس

فروشگاه آنلاین لوکس، کاملاً فارسی و راست‌به‌چپ، ساخته‌شده با **Next.js 14 (App Router)**،
**Tailwind CSS** و **Framer Motion** — که مستقیماً از REST API ووکامرس شما محصولات،
دسته‌بندی‌ها و تصاویر واقعی را می‌خواند.

## راه‌اندازی (۳ قدم)

### ۱. نصب پکیج‌ها
```bash
npm install
```

### ۲. اتصال به ووکامرس
یک فایل به اسم `.env.local` در ریشه‌ی پروژه بسازید (از روی `.env.local.example` کپی کنید) و پر کنید:

```
WORDPRESS_URL=https://yourdomain.com
WC_CONSUMER_KEY=ck_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
WC_CONSUMER_SECRET=cs_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

کلید و سکرت را از پنل وردپرس بسازید:
**WooCommerce → Settings → Advanced → REST API → Add key** (دسترسی روی «Read» کافیست).

### ۳. اجرا
```bash
npm run dev
```
آدرس [http://localhost:3000](http://localhost:3000) را باز کنید.

## چه چیزی از ووکامرس واقعی می‌آید؟

| بخش سایت | منبع داده |
|---|---|
| مجموعه‌های ویژه (صفحه‌ی اصلی) | ۳ دسته‌بندی برتر ووکامرس (بر اساس تعداد محصول) |
| پرفروش‌ترین‌ها (صفحه‌ی اصلی) | محصولات با فلگ «Featured» در ووکامرس |
| تازه‌ترین‌ها (صفحه‌ی اصلی) | جدیدترین محصولات بر اساس تاریخ انتشار |
| صفحه‌ی محصولات + دسته‌بندی‌ها | مستقیماً از `/wc/v3/products` و `/wc/v3/products/categories` |
| صفحه‌ی اختصاصی هر محصول | از `/wc/v3/products?slug=...` + محصولات مرتبط از همان دسته |
| تصاویر، قیمت، موجودی، مشخصات فنی (attributes) | همگی مستقیماً از ووکامرس |

## ⚠️ نکات مهم قبل از رفتن به تولید

1. **واحد پول**: اگر فروشگاه ووکامرس‌تان قیمت را به **ریال** ذخیره می‌کند نه تومان،
   در فایل `lib/adaptProduct.ts` مقدار `TOMAN_DIVIDER` را از `1` به `10` تغییر دهید.

2. **مقیاس مجسمه (Scale)**: کد دنبال یک ویژگی (attribute) به اسم «مقیاس» یا `scale`
   در محصول ووکامرس می‌گردد. اگر در ووکامرس چنین ویژگی‌ای نساخته‌اید، مقدار پیش‌فرض
   «استاندارد» نمایش داده می‌شود — می‌توانید این attribute را در ووکامرس اضافه کنید.

3. **نظرات محصول**: تب «نظرات» در صفحه‌ی محصول فعلاً به نظرات واقعی ووکامرس وصل نیست
   (فقط تعداد را نشان می‌دهد). برای اتصال کامل باید اندپوینت `/wc/v3/products/reviews`
   را هم اضافه کنیم — اگر لازم دارید بگویید تا اضافه شود.

4. **پرداخت و ثبت سفارش**: سبد خرید و صفحه‌ی تسویه حساب فعلاً سفارش را در
   `localStorage` مرورگر ذخیره می‌کنند (برای دمو/نمایش). برای ثبت سفارش واقعی در
   ووکامرس، باید فرم تسویه حساب به اندپوینت `/wc/v3/orders` وصل شود (این کار درگاه
   پرداخت واقعی مثل زرین‌پال را هم می‌طلبد).

5. **احراز هویت**: ورود/ثبت‌نام فعلاً یک سیستم دموی محلی (localStorage) است، نه
   حساب‌های واقعی وردپرس/ووکامرس. برای اتصال واقعی باید به JWT Authentication یا
   اندپوینت مشتریان ووکامرس (`/wc/v3/customers`) وصل شود.

## صفحات

| مسیر | توضیح |
|---|---|
| `/` | صفحه‌ی اصلی — همه‌چیز از ووکامرس زنده می‌آید |
| `/products` | لیست محصولات با فیلتر دسته‌بندی دایره‌ای (زنده از ووکامرس) + جست‌وجو |
| `/products/[slug]` | صفحه‌ی اختصاصی هر محصول (زنده از ووکامرس) |
| `/cart` | سبد خرید |
| `/checkout` | فرم آدرس + روش پرداخت + خلاصه‌ی سفارش |
| `/checkout/success` | تأییدیه‌ی سفارش |
| `/login` و `/register` | ورود و ثبت‌نام (دمو) |
| `/account` | اطلاعات حساب + تاریخچه‌ی سفارش‌ها |
| `/contact` | تماس با ما |

## ساختار پروژه

```
app/
  page.tsx                        — صفحه‌ی اصلی
  layout.tsx                       — فونت‌ها (Vazirmatn + Noto Naskh Arabic)، RTL
  providers.tsx                     — Auth + Cart + Toast context
  api/
    products/route.ts                — لیست محصولات (پراکسی امن به ووکامرس)
    products/[slug]/route.ts          — یک محصول بر اساس slug
    categories/route.ts                — لیست دسته‌بندی‌ها
  products/page.tsx                     — صفحه‌ی محصولات
  products/[slug]/page.tsx               — صفحه‌ی محصول
  cart/, checkout/, checkout/success/,
  login/, register/, account/, contact/    — بقیه‌ی صفحات
components/
  ProductGridSection.tsx    — گرید محصول زنده (برای پرفروش‌ها/تازه‌ها)
  ProductCard.tsx, ProductVisual.tsx  — کارت محصول + نمایش تصویر واقعی
  Navbar.tsx, Footer.tsx, ...
lib/
  woocommerce.ts     — اتصال امن سمت سرور به REST API ووکامرس
  adaptProduct.ts      — تبدیل داده‌ی خام ووکامرس به تایپ داخلی پروژه
  types.ts, cart-context.tsx, auth-context.tsx, toast-context.tsx, format.ts
```
